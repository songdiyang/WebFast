关于 WebFast

为什么创建 WebFast？

当前主流前端框架（React、Vue、Angular）虽然功能强大， 但与 AI 代码生成存在严重不适配：

  • 语法糖/DSL — JSX、Vue Template 等混合语法让 AI 混淆 HTML 与 JS 边界
  • 构建工具链 — AI 需理解 Webpack/Vite/Babel 配置，错误频发
  • 转译断层 — 写的代码 ≠ 运行的代码，调试困难
  • 抽象层过重 — Hooks 依赖数组、响应式系统常让 AI 误用

设计原则

零构建 / 零转译

浏览器原生 ES Modules，写什么就跑什么

标准 Web API

Web Components、Fetch、History API、CustomEvent

约定优于配置

严格的文件结构和命名规范，AI 只需按模板填充

代码 1:1 映射

AI 写的代码 = 人类写的代码 = 浏览器运行的代码
